1. Identificação do Projeto 
Nome do Projeto: Desenvolvimento de um Sistema de Gestão Financeira para Pequenas Empresas.
Autor: Pedro Rolin Schmitz
Orientador: Indefinido
Data: Não aprovado

2. Introdução e Visão Geral 
Este projeto visa desenvolver um aplicativo mobile para ajudar estudantes a organizarem seus estudos de forma prática. 
A solução centraliza tarefas, calendário, progresso acadêmico e integração com o Moodle, resolvendo a dificuldade de manter o 
controle do aprendizado em ferramentas separadas.

3. Objetivos do Projeto
Criar um app intuitivo para organização de estudos.
Implementar tarefas, calendário e progresso de aprendizado.
Integrar com o Moodle para sincronização de atividades.
Garantir usabilidade e compatibilidade com dispositivos móveis.

4. Escopo do Projeto
Entregáveis:
Aplicativo mobile com módulos de organização de tarefas, calendário, progresso de estudos e integração básica ao Moodle.
Documentação técnica completa do sistema.
Relatórios de testes e validação com usuários.

Requisitos:
Funcionais: 
Cadastro e gerenciamento de tarefas e eventos acadêmicos.
Integração com o Moodle para sincronização de disciplinas e atividades.
Acompanhamento de progresso individual.

Não Funcionais: 
Interface amigável e responsiva.
Desempenho estável em dispositivos Android.
Segurança básica na manipulação de dados pessoais.
 
Exclusões:
Integração com outras plataformas além do Moodle (como Google Classroom, Microsoft Teams).
Suporte técnico contínuo após entrega do projeto.
 
5. Recursos Necessários
Pessoas: Desenvolvedor e pesquisador (Pedro Rolin Schmitz), orientador (indefinido). 
 
Tecnologia: 
Ambiente de desenvolvimento (VSCode).

Disponibilidade do suporte do orientador.
Acesso a ferramentas de desenvolvimento e aprendizado.

8. Restrições 
Prazo final para a conclusão do TCC.
Orçamento limitado para aquisição de ferramentas ou licenças.
Dependência da disponibilidade do Moodle para integração e testes.

9. Riscos e Planos de Contingência 
Risco: Dificuldades técnicas na integração com o Moodle.
Mitigação: Pesquisar previamente a documentação da API do Moodle; considerar uso de dados simulados caso necessário.
Risco: Atrasos no cronograma de desenvolvimento.
Mitigação: Priorizar as tarefas essenciais e, se necessário, reduzir o escopo de funcionalidades menos críticas.
Risco: Limitação na performance em dispositivos móveis.
Mitigação: Testes contínuos em diferentes aparelhos durante o desenvolvimento.

10. Critérios de Aceitação  
O software atende a todos os requisitos funcionais definidos.
O aplicativo deve permitir o gerenciamento de tarefas, calendário e progresso de estudo.
A integração com o Moodle deve funcionar de forma básica e estável.
O sistema deve ser funcional em dispositivos móveis Android.
A interface deve ser intuitiva e validada com usuários finais.
